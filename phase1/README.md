# Phase 1 — ANE Residency Tooling

Measurement and conversion tooling for Phase 1 ANE-first execution on Mac Mini M4 24 GB.

**Strategy doc:** [`docs/Phase1_ANE_First_Strategy.md`](../docs/Phase1_ANE_First_Strategy.md)  
**Program status:** [`docs/OSLab_Program_Board.md`](../docs/OSLab_Program_Board.md)  
**Model I/O contracts:** [`phase1/NOTES.md`](NOTES.md)

## Files

| File | Purpose |
|------|---------|
| `coreml_convert.py` | Prefill-only HF → Core ML (no KV; `torch.export`) |
| `coreml_kv_convert.py` | Prefill-kv + MLState decode + `torch.export` decode (`--mode decode_torch_export`) |
| `coreml_quantize.py` | Post-export int4/int8 linear weight quantization |
| `kv_decode.py` | Stateful decode loops (MLX + Core ML prefill/decode hand-off) |
| `coreml_instrumentation.py` | `ComputeUnit` loading, metadata + `MLComputePlan` logging |
| `ane_residency_benchmark.py` | MLX vs Core ML benchmark with powermetrics + JSONL |
| `ane_placement_profile.py` | Short decode profiling + compute-plan capture |
| `NOTES.md` | I/O shapes, export history, ANE placement diagnosis |
| `requirements.txt` | Pinned deps for conversion venv (Python 3.12) |

## Setup

```bash
cd /path/to/alala
uv venv phase1/.venv --python 3.12
uv pip install --python phase1/.venv/bin/python -r phase1/requirements.txt
```

Harness benchmarks use repo `.env` for `SUDO_PASSWORD` (powermetrics) and `MLX_PYTHON` (MLX decode subprocess).

Converted `.mlpackage` files are **gitignored** (`models/*.mlpackage/`). Regenerate locally. TorchScript fallback `models/qwen2.5-0.5b-decode-kv.pt` is generated by `coreml_kv_convert.py` and kept local (not tracked).

## Export KV models

```bash
# Prefill + stateful decode (MLState default)
phase1/.venv/bin/python phase1/coreml_kv_convert.py \
  --output-dir models --max-ctx 1024

# MLState decode only (after prefill-kv exists)
phase1/.venv/bin/python phase1/coreml_kv_convert.py \
  --mode decode --output-dir models --max-ctx 1024

# torch.export decode (explicit KV I/O, ATEN dialect — recommended for ANE placement)
phase1/.venv/bin/python phase1/coreml_kv_convert.py \
  --mode decode_torch_export --output-dir models --max-ctx 1024
```

Artifacts: `models/qwen2.5-0.5b-prefill-kv.mlpackage`, `models/qwen2.5-0.5b-decode-kv.mlpackage` (MLState), `models/qwen2.5-0.5b-decode-kv-torch-export.mlpackage` (torch.export).

## Run ANE residency benchmark

```bash
# MLX real decode (KV inside mlx_lm)
python3 phase1/ane_residency_benchmark.py --backend mlx --decode --context 512,1024

# Core ML stateful decode (prefill-kv + decode-kv MLState)
phase1/.venv/bin/python phase1/ane_residency_benchmark.py \
  --backend coreml --decode --context 512 \
  --compute-units all

# Short profile mode (20s step, ctx 512)
phase1/.venv/bin/python phase1/ane_residency_benchmark.py \
  --backend coreml --decode --profile

# Legacy prefill-only proxy (no KV)
phase1/.venv/bin/python phase1/ane_residency_benchmark.py \
  --backend coreml --model models/qwen2.5-0.5b-ane.mlpackage --context 512,1024
```

`--compute-units` choices: `all`, `cpu_and_ne`, `cpu_and_gpu`, `cpu_only`. Benchmark logs load config and writes `results/ane_residency/<run_id>/coreml_load_report.json`.

## ANE placement profiling

```bash
PYTHONPATH=phase1 phase1/.venv/bin/python phase1/ane_placement_profile.py \
  --decode-kv models/qwen2.5-0.5b-decode-kv-torch-export.mlpackage \
  --max-ctx 1024 --compute-units all --context 512 --profile-tokens 30
```

Outputs: `results/ane_placement_profile/<run_id>/profile_report.json`, `compute_plan.json`, `logs/ane_placement_profile_<cu>_ctx<ctx>.powermetrics.txt`.

| Core ML | 1024 | 3.9 | **11.7%** |

Run: `ane_residency_20260701T002500Z_d1b410d0`

### Stateful decode

| Backend | Decode runtime | ctx | Sust. tok/s | ANE proxy |
|---------|----------------|-----|-------------|-----------|
| MLX | mlx_lm | 512 | 106.7 | ~0% |
| Core ML | TorchScript `.pt` | 512 | 35.0 | 0.3% |
| Core ML | **MLState `.mlpackage`** | 512 | 7.45 | **0.11%** |
| Core ML | torch.export fp16 | 512 | 7.93 | 0.067% |

**ANE placement:** torch.export fp16 **44.8% plan** / 0.07% runtime; **int4 44.1% plan / 2.9% runtime** — quant closes planner/runtime gap. See `NOTES.md`.

## Thermal safety

Default abort threshold: **85 °C** (`--temp-threshold`). Decode mode defaults to **88 °C**. Allow 60–90 s cooldown between context steps.
